# 不会动文件的 AI 扫描器

本地文件扫描工具：输入一个文件夹路径，扫描文件信息和文本内容，生成整理建议报告、执行计划和可检查脚本。

只给建议，不会移动、删除、重命名任何原文件。

## 运行方式

### 桌面界面（推荐）

```powershell
py -3 app_native.py
```

功能：
- 拖放或选择文件夹
- 选择扫描场景（通用 / 下载整理 / 工作文档 / 学习资料 / 照片整理）
- **生成 AI 整理建议**：勾选后填入 API Key、Base URL、模型名，扫描完成后生成整理优先级、风险提醒和步骤建议
- **取消扫描**：扫描过程中可随时取消
- **📊 可视化**：扫描完成后点击按钮查看图表（分类饼图、空间柱状图、健康分分布、可释放空间）
- 实时进度显示
- 手风琴式结果展示（待办清单、垃圾缓存、健康评分、分类统计、重复文件、整理建议、完整报告）
- 打开文件位置 / 复制路径

### 命令行

```powershell
py -3 organize.py “D:\Downloads”
```

运行完成后，在被扫描目录下生成：

```text
file-organization-report.md
file-organization-plan.md
delete-duplicates.bat
cleanup-candidates.bat
```

`.bat` 脚本不会自动运行，请先检查内容再手动执行。

## AI 整理建议

桌面界面支持勾选“生成 AI 整理建议”，填入：

- **API Key**：OpenAI 或兼容服务的密钥
- **Base URL**：默认 `https://api.openai.com/v1`，可改为 DeepSeek、Ollama、本地兼容服务等
- **模型**：默认 `gpt-4o-mini`

扫描和分类仍然在本地完成。AI 不再逐个文件分类，而是在扫描结束后，根据扫描摘要生成整理建议，包括优先处理事项、重点区域、风险提醒和建议步骤。

发送给 AI 的内容只包含扫描摘要：文件数量、总大小、分类统计、健康分、重复文件数量、可清理空间、少量文件名样例和问题摘要。不发送完整文件内容，不发送文件正文预览。

API Key、Base URL、模型名只存在当前运行时内存里，不会写入报告、计划、脚本或日志。报告中只保存 AI 返回的整理建议正文。

## 可视化图表

扫描完成后点击 **📊 可视化** 按钮，弹出图表窗口，包含：

- 文件分类分布饼图
- 各分类占用空间柱状图
- 健康评分分布直方图
- 可释放空间统计（重复文件 / 低风险清理 / 中风险清理）

需要 `matplotlib`：`pip install matplotlib`

## 安全边界

工具只写入这些生成文件，不会自动执行脚本，也不会移动、删除、重命名原文件。

使用 AI 整理建议时，API Key、Base URL、模型名不会保存到任何生成文件。请注意：扫描摘要会发送给你填写的 API 服务，因此如果文件名本身包含敏感信息，建议不要启用 AI，或改用可信的本地/内网兼容 API。

## 分类规则

离线模式根据扩展名、文件名关键词、文本预览判断分类。内置分类：文档、代码、图片/截图、压缩包、合同、财务/发票、学习笔记、日志、其他。

之前的 AI 分类和离线分类差别不明显，是因为离线规则已经优先按扩展名和关键词覆盖常见文件，而 AI 也只能返回同一套分类。现在 AI 改为整理顾问，关注全局优先级、风险和步骤建议，和本地分类结果形成互补。

性能优化：
- 文本预览只读前 4KB
- 重复检测先按大小分组，只对候选文件计算 SHA256
- 空文件跳过 SHA256 计算
- 扫描深度限制 20 层，跳过隐藏目录
