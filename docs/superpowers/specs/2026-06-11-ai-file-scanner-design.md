# 不会动文件的 AI 扫描器 - 设计规格

## 目标

做一个本地命令行工具：用户输入一个文件夹路径，工具只扫描文件信息和可读取的少量文本内容，在该目录下生成 `file-organization-report.md` 整理建议报告。

第一版必须安全：除覆盖生成报告文件外，不移动、不删除、不重命名任何原文件。

## 推荐方案

采用单文件 Python 脚本 `organize.py`。

选择理由：第一版范围小，标准库足够完成扫描、分类、哈希和 Markdown 报告生成。单文件更容易运行和理解，后续功能变多时再拆分模块。

## 命令行行为

运行方式：

```bash
python organize.py "目标文件夹路径"
```

行为：

- 校验目标路径存在且是文件夹。
- 递归扫描目标文件夹中的文件。
- 跳过本工具生成的 `file-organization-report.md`，避免报告把自己纳入扫描。
- 生成或覆盖目标目录下的 `file-organization-report.md`。
- 遇到单个文件无法读取时，把错误写入报告，不中断整体扫描。

## 组件边界

`organize.py` 内保留清晰函数/类边界：

- `FileInfo`：保存文件路径、大小、扩展名、修改时间、哈希、文本预览等元数据。
- `scan_directory()`：遍历目录并收集文件信息。
- `read_text_preview()`：只读取文本类文件的少量内容。
- `Classifier`：分类器接口边界。
- `OfflineRuleClassifier`：第一版离线规则分类器。
- `suggest_name()`：给出更规整的重命名建议文本。
- `find_duplicates()`：基于大小和 SHA256 找可能重复文件。
- `write_report()`：生成 Markdown 报告。

## 分类规则

第一版使用离线规则，不联网，不需要 API key。

分类依据包括：

- 文件扩展名。
- 文件名关键词。
- 文本预览关键词。
- 文件大小。
- SHA256 哈希。

初始分类包括：

- 文档
- 代码
- 图片/截图
- 压缩包
- 合同
- 财务/发票
- 学习笔记
- 日志
- 其他

## 报告内容

`file-organization-report.md` 使用简体中文，至少包含：

1. 扫描概览：扫描路径、文件数量、总大小、生成时间。
2. 建议移动：按分类列出建议放入的文件夹。
3. 建议重命名：列出原文件名和建议文件名。
4. 可能重复：列出哈希相同的文件组。
5. 读取失败：列出无法读取或处理的文件及原因。
6. 安全说明：明确本工具没有移动、删除、重命名任何原文件。

## 错误处理

- 目标路径不存在或不是文件夹：命令行报错并退出，不生成报告。
- 单个文件权限不足或读取失败：记录到报告，继续扫描其他文件。
- 二进制文件不做内容解析，只用文件名、扩展名和大小判断。

## 测试标准

使用临时测试目录验证：

- 能生成 `file-organization-report.md`。
- 报告包含分类建议、重命名建议、重复文件和安全说明。
- 除报告文件外，测试目录中的原文件内容和文件名不变。
- 旧报告存在时会被覆盖。

## 非目标

第一版不做：

- 自动移动文件。
- 自动删除文件。
- 自动重命名文件。
- OCR 图片识别。
- PDF/docx 深度解析。
- Web 界面。
- 联网 AI 调用。
