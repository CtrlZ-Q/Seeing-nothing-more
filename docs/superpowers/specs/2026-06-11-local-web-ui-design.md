# 本地网页界面 - 设计规格

## 目标

在现有离线文件扫描器上增加一个本地网页界面。用户运行本地 Python 服务后，在浏览器中输入文件夹路径，点击扫描，页面完整显示生成的整理建议报告。

安全边界不变：除覆盖目标目录下的 `file-organization-report.md` 外，不移动、不删除、不重命名、不修改任何原文件。

## 推荐方案

新增单文件 `web_app.py`，只使用 Python 标准库。

选择理由：当前项目没有依赖管理，也希望用户直接运行。标准库的 `http.server` 足够支持本地表单、POST 请求和结果展示，避免安装 Flask 等依赖。

## 运行方式

```powershell
py -3 web_app.py
```

默认监听：

```text
http://127.0.0.1:8765
```

## 页面行为

- 首页显示一个文件夹路径输入框。
- 用户输入本机文件夹路径后点击“开始扫描”。
- 后端复用 `organize.py` 中的扫描、分类、重复检测和报告生成逻辑。
- 扫描完成后，页面完整显示 `file-organization-report.md` 的 Markdown 文本内容。
- 页面同时显示报告文件路径。

## 后端路由

- `GET /`：返回 HTML 页面。
- `POST /scan`：读取表单中的 `folder` 字段，执行扫描并返回结果页。

## 错误处理

- 路径为空：页面显示错误。
- 路径不存在：页面显示错误。
- 路径不是文件夹：页面显示错误。
- 单个文件读取失败：沿用 `organize.py` 的行为，记录到报告，不中断整体扫描。

## 测试标准

- `py -3 -m py_compile web_app.py organize.py` 通过。
- 启动本地服务后，`GET /` 返回表单页面。
- `POST /scan` 对临时测试目录返回完整报告内容。
- 运行前后临时测试目录原文件哈希一致。
- 测试后清理临时文件和后台服务进程。

## 非目标

第一版网页界面不做：

- 用户登录。
- 外网访问。
- 文件上传。
- 自动打开系统文件选择器。
- 实时进度条。
- 把 Markdown 转成复杂 HTML 样式。
- 自动移动、删除、重命名文件。
