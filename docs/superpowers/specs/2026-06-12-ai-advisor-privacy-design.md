# AI 整理顾问与隐私边界设计

## 目标

把现有“AI 分类”替换为“AI 整理顾问”。AI 不再逐个文件参与分类，避免和离线规则结果高度相似；扫描仍使用本地规则完成分类、重复文件、清理候选和报告生成，AI 只在扫描完成后基于摘要生成整理建议。

## 安全与隐私边界

- API Key、Base URL、模型名只存在当前运行时内存中。
- 不把 API Key、Base URL、模型名写入报告、计划、脚本、日志或说明书生成结果。
- API Key 输入框使用密码显示。
- 工具不会自动移动、删除、重命名文件，也不会自动执行生成的 `.bat` 脚本。
- AI 整理顾问会把扫描摘要发送给用户填写的 API 服务：总文件数、总大小、分类统计、健康分、可清理空间、重复文件数量、少量文件名样例和问题摘要。
- 不发送完整文件内容，不发送文件正文预览，不发送 API Key/Base URL/模型名作为提示词内容。

## 功能设计

界面把“启用 AI 分类”改为“生成 AI 整理建议”。用户填写 API Key、Base URL 和模型名后点击扫描。扫描流程先照常使用离线规则完成本地分析；扫描结束后，如果启用 AI 整理建议，则额外请求一次 AI，让它输出：

1. 最优先处理的 3-5 件事。
2. 哪些类型文件最值得整理。
3. 重复文件和缓存清理的风险提醒。
4. 建议的整理步骤。

AI 建议会显示在界面结果区，并写入本地报告中的“AI 整理建议”章节。报告只保存建议正文，不保存 API 配置。

## 为什么替换 AI 分类

现有离线分类已经按扩展名和关键词覆盖常见文件；AI 分类又被限制只能返回同一套分类，因此大多数常见文件结果会和离线模式接近。AI 整理顾问使用的是全局摘要视角，能给出优先级、风险和行动步骤，和离线分类有明显区别，也更符合“扫描后帮我决定怎么整理”的用途。

## 错误处理

如果 AI 请求失败，扫描结果仍然保留；界面显示“AI 整理建议生成失败”，报告不写入 API 配置。失败不影响离线报告、计划和脚本生成。

## 验证

- 搜索确认 API Key、Base URL、模型名没有写入任何报告生成逻辑。
- 检查 AI 提示词只包含摘要和少量样例，不包含完整文件内容。
- 运行 Python 语法检查。
- 手动扫描一个测试文件夹，确认不开 AI 时保持离线结果，开 AI 时多出“AI 整理建议”。
